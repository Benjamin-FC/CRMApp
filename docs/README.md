## Documentation Standards

All documentation follows these principles:
- **Clear and Concise** - Easy to understand and follow
- **Step-by-Step** - Detailed instructions with examples
- **Up-to-Date** - Regularly updated with code changes
- **Searchable** - Well-organized with clear headings and structure

## Contributing to Documentation

When adding new documentation:
1. Place design documents in `design/`
2. Place deployment documents in `deployment/`
3. Place general guides in the root `docs/` folder
4. Update this README with links to new documents
5. Use clear, descriptive filenames in UPPERCASE with hyphens
